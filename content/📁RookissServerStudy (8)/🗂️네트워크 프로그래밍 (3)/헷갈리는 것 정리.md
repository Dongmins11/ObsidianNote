
커넥터와 리스너의
소켓차이


### 📡 1️⃣ 클라이언트 측

```csharp
Socket socket = new Socket(AddressFamily.InterNetwork, SocketType.Stream, ProtocolType.Tcp); 

socket.ConnectAsync(args);
```

- 여기서 클라이언트는 스스로 소켓을 **직접 생성**함.
    
- `ConnectAsync()` 호출 시, OS 커널이 내부적으로 서버와의 TCP **3-way handshake** (SYN → SYN+ACK → ACK)을 수행.
    
- 연결이 완료되면 이 소켓(`ConnectSocket`)은 서버와 직접 연결된 상태가 됨.
    

즉, 클라이언트는 **“내가 만든 소켓으로 서버에 전화를 건 것”**.



### 🖥️ 2️⃣ 서버 측

`_listenSocket.AcceptAsync(args);`

- 서버는 `new Socket()` 을 직접 하지 않아.  
    `_listenSocket`(리스닝 소켓)은 **“전화 받을 준비만 하는 문지기”** 역할임.
    
- 클라이언트가 접속을 시도하면, OS 커널이  
    “이 클라이언트와 통신할 **새 소켓(통신 전용)**”을 **자동으로 생성**해서  
    `args.AcceptSocket`에 담아줌.
    
- 즉, 서버는 매번 “새로운 클라이언트 1명당 소켓 1개”를 받게 되는 거야.


## 🔄 요약 비교 표

|구분|클라이언트(Connector)|서버(Listener)|
|---|---|---|
|소켓 생성|직접 `new Socket()`|OS가 `Accept()` 시 자동 생성|
|소켓 변수|`ConnectSocket`|`AcceptSocket`|
|용도|서버와 연결하기 위한 통신선|클라이언트와 통신하기 위한 통신선|
|생성 시점|연결 시도 직전|클라이언트 접속 완료 시|
|개수|1개 (서버 1곳당)|클라이언트 수만큼 생성됨|
|소유권|클라이언트 프로세스|서버 프로세스|
|연결 방향|outbound (outgoing)|inbound (incoming)|

---

## 💬 한 줄 비유로 마무리

- **클라이언트:** “내가 전화기를 만들어서 서버 번호로 전화를 건다.” ☎️
    
- **서버:** “OS가 전화가 오면 자동으로 새 수화기를 만들어 나에게 건네준다.” 📞
    

즉,  
👉 서버의 `AcceptSocket`은 OS가 만들어준 **새 통신선**,  
👉 클라이언트의 `ConnectSocket`은 직접 만든 **전화선**이야.
