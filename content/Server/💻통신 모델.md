### 현실 세계 예시
1. 상품 포장
2. 배송 정책 (택배만 문앞에 툭 놓는 경우도 있지만 중요한 택배는 수취인 확인하는 방식도 있음)
3. 최종 주소  ->(from , to 경로)
4. 단지 경로  ->(다음으로 이 택배를 전달해야 할 정보) (예 : 201호 -> 경비실 -> 택배 배송센터로 보면 지금 화살표 방향에 대한 위치를 말하는 것 현재 위치에서 다음 방향으로 가는 경로 위치라 보면됨)
5. 택배 운송 -> 택배를 운송하는 것

### 네트워크 예시
1. 어플리케이션  -> HTTP, FTP, DNS
2. 트랜스포트  -> 전송 확인 / 오류해결 정책  (TCP, UDP관련해서 이후 제대로 설명)
3. 네트워크  -> 네트워크 간 경로 설정 (IPv4, IPv6)  관련 장치 : 라우터
4. 데이터 링크  -> 네트워크 내 경로 설정 (이더넷, PPP)  관련 장치 : 스위치
5. 피지컬  -> 신호 처리  관련 장치: 케이블/허브

---
### 주소는 크게 3가지로 나뉨

1. 7 계층 주소 (도메인)_: 도메인 주소를 숫자로 표현하기엔 보기 불편해서 문자로 표현  예 : www.naver.com  나중에 네트워크 주소로 변환해서 하는걸 알아보겠다 
2. 3계층 주소 (IP) 3계층 (장치 : 라우터)
3. 2계층 주소 (MAC) 기기에 생성될 때 네트워크 카드에 박혀있는 주소 (장치 : 스위치)  

### 왜 굳이 계층을 분리해서 생각을하냐?
만약 인터넷이 안되서 수리기사님을 불렀다
수리기사님은 저 계층 하나하나 검사해보면서 어디가 고장났는지 확인을해본다
즉, 책임을 명확하게 두고 처리를 해야 보다 간편하게 수리하거나 유지보수 할 수 있다

### 모델을 왜 정하냐?
이걸가지고 표준 통신약속을 잡는것 이다  
예를들어 여러 네트워크 기기를 만드는 회사가 자기만의 통신약속을 잡아버리면  
그 회사만의 통신약속을 알고 있는 기기만 통신이 가능하다는 얘기다  
그래서 표준 약속을 정해두고 그걸 토대로 통신을 하는것이다.
